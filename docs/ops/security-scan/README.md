# Supply Chain Security Baseline

This directory stores machine-readable evidence generated by the repository-local `supply-chain-security` workflow.

## Policy Summary

- Dependabot is enabled per repository for the local package ecosystem and for GitHub Actions.
- Open findings with `CVSS >= 7.0` block the `supply-chain-security` check unless a time-bounded approved exception exists.
- GitHub Advisory CVSS is the primary score source.
- NVD CVSS is used only when the GitHub Advisory score is missing.
- Findings with no GitHub Advisory or NVD score stay in `manual-triage-required` until a reviewer decision is recorded in `.github/security/dependency-exceptions.json`.
- GitHub Actions `uses:` references must be pinned to full-length commit SHAs.

## Exception Registry

- Canonical mutable source: `.github/security/dependency-exceptions.json`
- Export snapshot: `docs/ops/security-scan/<YYYYMMDD>/<snapshot-id>/exceptions-<repo>.json`

Each approved record must include:

- `id`
- `recordType`
- `scope`
- `match`
- `decision`
- `owner`
- `reason`
- `evidence`
- `requestedAt`
- `reviewedAt`
- `reviewer`
- `expiresAt`
- `status`

## Required Secrets

- `DEPENDABOT_ALERTS_TOKEN`
  Use if `GITHUB_TOKEN` cannot read Dependabot alerts for the repository.
- `BRANCH_PROTECTION_TOKEN`
  Required to capture live branch-protection and GitHub Actions permissions evidence. This token needs repository administration access.
  Repository workflow runs fail closed until this proof can be captured.

## Artifact Contract

Each workflow run writes evidence under `docs/ops/security-scan/<YYYYMMDD>/<snapshot-id>/`:

- `index.json`
- `scan-summary-<repo>.json`
- `branch-protection-<repo>.json`
- `exceptions-<repo>.json`
- `dependabot-alerts-<repo>.json`

The `index.json` file is repository-local and lists only the current repository in `repositories[]`.

The default `snapshot-id` is `github.run_id-github.run_attempt` in GitHub Actions and a UTC timestamp for local/manual execution. This keeps multiple runs on the same day audit-traceable instead of overwriting the previous snapshot.

## Local Validation

npm run lint:supply-chain
